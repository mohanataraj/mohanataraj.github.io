2.2 Mapping Content & Interaction

2.2 Mapping Content and Interaction

This page is mostly dedicated to writing content mapping and interaction via flowchart or sequence diagram using mermaid. For more advanced we can use other online editor’s such as draw.io

2.2.1 Mapping Content:

In this section we focus only on developing the content on each page of the app.

We follow the rules defined below for defining our content mapping.

RULES:

  • we use flowchart graphs for different pages.
  • subgraphs to map each content of a particular page - if it requires anything.
  • arrow heads represents moving to different page.
  • without arrow head represents line in the same page.
  • color codes for different elements of the flowchart will evolve as we work.
Note: more rules may be added if and when required to better represent this phase of our work.
2.2.2.1 Major components of App:

The 4 major components of the applications are the following:

  1. Welcome page - contents
  2. Food
  3. Exercise
  4. Wellness - (Mood/Stress level)

1. Welcome Page:

This is the page when the user enters, sees the welcome message and statistics for the day.

The botton nav bar of the application is designed to make sure the objective of the user is reached within 3-clicks, such as to go to any aspect of the application - view their stats, create challenges, blog, create events, find events etc…
